# Logback

Logback官方文档open in new window

# 介绍

Logback和Log4j是同一个作者，Logback目前分为三个模块，分别为 logback-core, logback-classic 和 logback-access.

  • logback-core: 为其他两个模块奠定了基础
  • logback-classic: 该模块可以被视为log4j 1.x的显著改进版本。此外，原生支持SLF4J API，使您可以方便地在logback、log4j 1.x或java.util.logging（JUL）等其他日志框架之间切换。
  • logback-access: 该模块与Servlet容器（如Tomcat和Jetty）集成，以提供HTTP访问日志功能。

日志级别分为TRACE, DEBUG, INFO, WARN, ERROR五个等级，各级别的优先级由低到高排序为：TRACE < DEBUG < INFO < WARN < ERROR。
通过在配置文件中指定某一个日志级别来控制系统要打印的内容。日志框架会打印当前指定级别的日志和比当前指定级别更严重的级别的日志。

  • 错误（ERROR）：表示程序运行出错，比如抛异常等情况。
  • 警告（WARN）：表示程序运行过程中有潜在风险，但此时并没有报错。
  • 信息（INFO）：表示程序运行过程中完成了一个关键动作，需要以程序运行信息的形式告知开发者。
  • 调试（DEBUG）：表示程序运行过程中更加细致的信息，协助程序员调试程序。
  • 跟踪 (TRACE) ：最低级别的日志，用于记录程序执行的详细信息和内部状态，通常用于排查问题。

# 快速入门

# 基本使用

导入依赖

<!-- 日志 会自动传递slf4j门面-->
<dependency>
    <groupId>ch.qos.logback</groupId>
    <artifactId>logback-classic</artifactId>
    <version>1.2.3</version>
</dependency>

编写配置(输出格式和输出级别) 我们需要把配置文件放在main/resources目录下，并配置文件命名为logback.xml

<?xml version="1.0" encoding="UTF-8"?>
<configuration debug="true">
    <!-- 指定日志输出的位置，ConsoleAppender表示输出到控制台 -->
    <appender name="STDOUT"
              class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <!-- 日志输出的格式 -->
            <!-- 按照顺序分别是：时间、日志级别、线程名称、打印日志的类、日志主体内容、换行 -->
            <pattern>[%d{HH:mm:ss.SSS}] [%-5level] [%thread] [%logger] [%msg]%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 设置全局日志级别。日志级别按顺序分别是：TRACE、DEBUG、INFO、WARN、ERROR -->
    <!-- 指定任何一个日志级别都只打印当前级别和后面级别的日志。 -->
    <root level="TRACE">
        <!-- 指定打印日志的appender，这里通过“STDOUT”引用了前面配置的appender -->
        <appender-ref ref="STDOUT" />
    </root>

    <!-- 根据特殊需求指定局部日志级别，可也是包名或全类名。 -->
    <logger name="com.atcode.mybatis" level="DEBUG" />

</configuration>

编写一个测试类，来测试一个日志的输出

public class LogTest {
    // 注意这里导包一定要注意 导的包为 org.slf4j 千万不要导错
    // 创建日志对象 并把当前类的对象传入 便于日志打印全类名
    private Logger logger = LoggerFactory.getLogger(LogTest.class);

    @Test
    public void test() {
        // 因为配置文件中配置的日志级别为TRACE最低的级别
        // 所以下面所有的级别都可输出
        logger.trace("trace");
        logger.debug("debug");
        logger.info("info");
        logger.warn("warn");
        logger.error("error");

        // 高级扩展
        String name = "root";
        String password = "123456";
        logger.info("账号：" + name + "密码:" + password);
        //优化
        logger.info("账号：{}, 密码：{}", name, password);
    }
}

# Lombok的使用-@Slf4j

在idea中安装lombok的插件并导入依赖

<!--lombok依赖-->
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.24</version>
</dependency>

这是我们只需在类上加上注解@Slf4j，就会自动给我们生成一个Logger的对象，名为log，我们就可直接使用

@Slf4j //log
public class LombokLogTest {
    @Test
    public void test() {
        // 我们直接可以使用注解为我们生成的Logger对象 log
        log.trace("trace");
        log.debug("debug");
        log.info("info");
        log.warn("warn");
        log.error("error");

        // 高级扩展
        String name = "root";
        String password = "123456";
        log.info("账号：" + name + "密码:" + password);
        //优化
        log.info("账号：{}, 密码：{}", name, password);
    }
}
